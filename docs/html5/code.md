# HTML5 高频面试题汇总

## 1. HTML5 有哪些新特性？

- **语义化标签**: `<header>`, `<footer>`, `<nav>`, `<section>`, `<article>`, `<aside>` 等。
- **增强型表单**: `input` 的新类型 (`date`, `email`, `url`, `number`, `range`, `color`) 和新属性 (`placeholder`, `required`, `pattern`, `autofocus`)。
- **音频和视频**: `<audio>` 和 `<video>` 标签。
- **Canvas 绘图**: 用于在网页上绘制图形。
- **SVG 绘图**: 可缩放矢量图形。
- **地理定位 (Geolocation)**: `navigator.geolocation` API。
- **拖放 API (Drag and Drop)**: `draggable` 属性和相关事件。
- **Web Storage**: `localStorage` 和 `sessionStorage`。
- **Web Worker**: 运行在后台的 JavaScript，不阻塞主线程。
- **WebSocket**: 全双工通信协议。

## 2. 什么是语义化？为什么要语义化？

**定义**：用正确的标签做正确的事情（如段落用 `p`，标题用 `h1-h6`，导航用 `nav`）。

**优点**：

1. **代码结构清晰**：易于阅读和维护。
2. **有利于 SEO**：搜索引擎爬虫依赖标签来确定上下文和各个关键字的权重。
3. **无障碍访问 (Accessibility)**：方便屏幕阅读器（Screen Reader）解析，帮助视障人士阅读。
4. **设备兼容性**：在没有 CSS 的情况下也能呈现良好的内容结构。

## 3. src 和 href 的区别

| 特性         | src (Source)                              | href (Hypertext Reference)              |
| :----------- | :---------------------------------------- | :-------------------------------------- |
| **含义**     | **引用资源**，替换当前元素                | **超文本引用**，建立连接                |
| **作用**     | 将指向的资源下载并应用到文档中            | 建立当前文档和外部资源的联系            |
| **加载行为** | **会暂停**其他资源的下载和处理 (同步加载) | **不会停止**对当前文档的处理 (并行下载) |
| **典型示例** | `<script src="...">`, `<img src="...">`   | `<link href="...">`, `<a href="...">`   |

**总结**：`src` 是“拿过来用”，`href` 是“连过去”。

## 4. script 标签中 defer 和 async 的区别

如果不加任何属性，script 会**立即下载并执行**，阻塞 HTML 解析。

| 属性       | 下载时机              | 执行时机                       | 执行顺序                        | 适用场景                |
| :--------- | :-------------------- | :----------------------------- | :------------------------------ | :---------------------- |
| **(默认)** | 阻塞 HTML 解析        | 下载完立即执行 (阻塞 HTML)     | 顺序执行                        | 关键且依赖 DOM 的脚本   |
| **async**  | **异步下载** (不阻塞) | 下载完**立即执行** (阻塞 HTML) | **不保证顺序** (谁先下完谁执行) | 独立的脚本 (如统计代码) |
| **defer**  | **异步下载** (不阻塞) | **HTML 解析完成后**执行        | **保证顺序** (按引入顺序执行)   | 依赖 DOM 或其他脚本的库 |

**图解**：

- `async`: Load (Async) -> Execute (Block) -> Render continues
- `defer`: Load (Async) -> Render finishes -> Execute

## 5. DOCTYPE 的作用是什么？

`<!DOCTYPE html>` 声明位于文档最前面，告诉浏览器使用哪种 **HTML 规范 (标准模式)** 来渲染页面。

- **如果不写**：浏览器会进入 **怪异模式 (Quirks Mode)**，模拟老式浏览器的行为，导致盒模型、布局等渲染与标准不一致（例如 IE 盒模型问题）。
- **HTML5**：只需要写 `<!DOCTYPE html>`，非常简洁。

## 6. 行内元素、块级元素、空(void)元素有哪些？

- **块级元素 (Block)**: 独占一行，可设置宽高。
  - `div`, `p`, `h1-h6`, `ul`, `ol`, `li`, `header`, `footer`, `section`, `table`, `form`
- **行内元素 (Inline)**: 不独占一行，宽高由内容决定，设置宽高无效（margin 上下无效）。
  - `span`, `a`, `label`, `b`, `strong`, `i`, `em`
- **行内块元素 (Inline-block)**: 不独占一行，但可设置宽高。
  - `img`, `input`, `button`, `select`, `textarea`
- **空元素 (Void)**: 没有闭合标签。
  - `br`, `hr`, `img`, `input`, `link`, `meta`

## 7. Canvas 和 SVG 的区别

| 特性         | Canvas                        | SVG                         |
| :----------- | :---------------------------- | :-------------------------- |
| **本质**     | **位图** (基于像素)           | **矢量图** (基于 XML)       |
| **渲染方式** | 脚本绘制 (JavaScript)         | DOM 节点 (HTML/XML)         |
| **事件处理** | 不支持 (只能监听整个 Canvas)  | **支持** (每个图形都是 DOM) |
| **缩放**     | 失真 (模糊)                   | **不失真**                  |
| **适用场景** | 游戏、大量数据可视化 (高性能) | 图标、地图、高保真图表      |

## 8. 介绍一下 Web Worker

JavaScript 是单线程的，大量计算会阻塞 UI 渲染。

**Web Worker** 允许我们在后台运行 JavaScript 脚本，**不干扰用户界面**。Worker 线程与主线程并行运行。

**限制**：

1. **同源限制**：Worker 脚本文件必须与主线程同源。
2. **DOM 限制**：Worker 无法读取主线程的 DOM (document, window, parent)。
3. **通信**：通过 `postMessage` 发送消息，`onmessage` 接收消息。

## 9. 浏览器存储方式的区别 (Cookie, localStorage, sessionStorage)

| 特性             | Cookie                           | localStorage                | sessionStorage                          |
| :--------------- | :------------------------------- | :-------------------------- | :-------------------------------------- |
| **数据生命周期** | 一般由服务器生成，可设置过期时间 | **永久有效** (除非手动删除) | **仅在当前会话有效** (关闭标签页即失效) |
| **存储大小**     | 4KB 左右                         | 5MB 左右                    | 5MB 左右                                |
| **与服务器通信** | **每次请求都会携带** (浪费带宽)  | 不参与服务器通信            | 不参与服务器通信                        |
| **易用性**       | 原生 API 难用 (需封装)           | `setItem`, `getItem`        | `setItem`, `getItem`                    |
| **用途**         | 维持 Session 状态 (Token)        | 长期保存用户偏好配置        | 临时保存表单数据                        |

## 10. title 和 alt 的区别

- **alt (Alternative)**: 图像的**替代文本**。当图片无法加载时显示，有利于 SEO 和无障碍阅读。
- **title**: 元素的**咨询信息**。当鼠标悬停在元素上时显示的提示文本 (Tooltip)。
